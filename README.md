# Upload simulation

## Abstract

CMSなどのアプリケーション作成時に、ファイルアップロードをする際に、ストレージへの書き込みとDBをinsertすることがある。それを非同期処理に埋め込むためのシュミレーションコード。

node.jsのイベントループや`Promise`などへの理解力を試すためにも実施。

## 基本的な処理

- ストレージアップロード後にDBへfilepath等をinsertする。
- ストレージアップロードに失敗したら、ストレージを削除。
- DBのinsertに失敗したらロールバック後にストレージのfileを削除。

## 留意事項

ストレージアップロードや、DBのinsertに関しては`setTimeout`等を使って、実際には処理を行うものとする。
